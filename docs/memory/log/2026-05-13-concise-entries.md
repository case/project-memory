# 2026-05-13: Entries are concise

Recent entries were running long, especially "Load-bearing decisions" sections where each bullet became a mini-essay.

## What landed

- `README.md` "Conventions": new bullet **Entries are concise**, after "Core files stay short."
- `AGENTS.md` and `init/templates/AGENTS.md`: matching one-line rule between the writing and naming rules.

## Why

Memory entries get code-reviewed. Shorter entries review better. "Bigger than commit, smaller than design doc" gives writers a felt sense of the target rather than an abstract "be concise."
